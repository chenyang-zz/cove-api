# codegen

`cmd/codegen` 是项目内置代码生成工具，扫描 Go 源码中的路由注解与 GORM 模型，自动生成：

- **Gin handler** — type 定义、构造函数、请求绑定方法
- **logic 骨架** — `internal/logic/<domain>/<component>.go`
- **类型安全仓储** — interface + PostgreSQL 实现 + BeforeCreate hook
- **OpenAPI 3.0 文档** — `docs/openapi.json`
- **类型安全提示词客户端** — `internal/prompts/promptsgen/<namespace>_gen.go`

生成产物分两类：repository 接口与 PostgreSQL 实现（`includeHeader=true`）以 `// Code generated by codegen; DO NOT EDIT.` 头标记；handler / logic 骨架（`includeHeader=False`）不加该头，而是通过格式化后比较实现幂等。两种产物工具均拒绝覆盖不匹配的非生成文件。

## 架构

### 生成管线

```
扫描（scan）
  ↓
  遍历 internal/transport/http/routes/*.go        → 路由注册调用
  遍历 internal/transport/http/request/*.go        → request DTO 结构
  遍历 internal/transport/http/response/*.go       → response DTO 结构
  遍历 internal/models/*.go                         → GORM 模型字段
  遍历 internal/transport/http/handler/*.go         → 已有 handler 方法
  遍历 internal/logic/**/*                          → 已有 logic 文件
  ↓
解析（parse）
  ↓
  route 注释组 → Directive（auth/userID/sse/event/input/output/description/summary/tag）
  Gin 路由注册（含 group 前缀、middleware 参数） → Route（method/path/handler/domain）
  request DTO tag → RequestDTO（json/form/query/uri/multipart）
  GORM 模型字段 → ModelInfo / ModelField
  ↓
渲染（render）
  ↓
  模板（.gotmpl） + 数据 → 格式化前的 Go 源码 / JSON
  ↓
写入（write）
  ↓
  format.Source → 与现有内容比较
    → 不存在     → FileAdded / FileWouldAdd
    → 内容一致   → FileUnchanged
    → 内容不同   → FileModified / FileWouldModify
    → 已存在     → FileSkipped
```

每步结果都累积到 `Report` 中，文件级变更与诊断级信息分别通过 `Files` 与 `Diagnostics` 字段暴露。

### 模板清单

| 模板 | 产物 |
|---|---|
| `handler_type.gotmpl` | `HandlerType` struct + 构造函数 |
| `handler_method.gotmpl` | 单个 handler 方法（含请求绑定、SSE 处理） |
| `logic.gotmpl` | `LogicType` struct + 构造函数 + 方法骨架 |
| `repository_interface.gotmpl` | `internal/repository/<model>.go` — 类型安全仓储接口 |
| `repository_postgres.gotmpl` | `internal/repository/postgres/<model>.go` — GORM 实现 |
| `model_hooks.gotmpl` | `internal/models/hooks.go` — `ensureUUID` 辅助函数 |
| `model_before_create.gotmpl` | `internal/models/hooks.go` — 单个模型的 `BeforeCreate` hook |

提示词生成不使用 `.gotmpl` 生成器模板：它直接解析 `internal/prompts/prompts.go`
中的 `registeredPrompts()` 和对应 Go template AST，再按 namespace 输出确定性的 Go 类型与方法。

### 文件保护

- **DO NOT EDIT header**：repository 接口与 PostgreSQL 实现（`includeHeader=true`）以 `// Code generated by codegen; DO NOT EDIT.` 开头。内容完全由工具维护，不应手动修改。
- **拒绝覆盖**：`writeGeneratedFile` 在写入前检查目标文件是否已存在且头部不含生成标记；若匹配失败，直接返回错误 `refuse to overwrite non-codegen file <path>`，避免误改手写文件（仅对 `includeHeader=true` 的产物生效）。
- **幂等性**：生成结果通过 `go/format.Source` 格式化后与磁盘文件比较；一致则标记 `FileUnchanged`，不触发写入。handler / logic 文件（`includeHeader=false`）不加 DO NOT EDIT 头，依靠 `appendGoFile` 与方法级追加实现可重复生成。

## 命令

### `route`

扫描 `internal/transport/http/routes/*.go`，为每个注册在自己 handler 组下的路由生成产物。

**产物**

| 产物 | 路径 | 说明 |
|---|---|---|
| handler type | `internal/transport/http/handler/<domain>.go` | 仅当文件不存在时生成；包含 `NewHandlerType` 构造函数 |
| handler method | 同上 | 追加到已有文件末尾；包含请求/URI/Query/Multipart 绑定、`UserID` 提取、logic 调用 |
| logic skeleton | `internal/logic/<domain>/<component>.go` | 仅当文件不存在时生成；文件名遵循缩写感知 snake_case |

**Flag**

| Flag | 默认值 | 说明 |
|---|---|---|
| `-root` | `.` | 项目根目录 |
| `--dry-run` | `false` | 预览 would-add / would-modify，不写入磁盘 |
| `--check` | `false` | 仅检查生成结果是否过期；有待变更时返回 `ErrCheckFailed`（exit code 1） |
| `--list` | `false` | 列出所有路由目标及其 handler/logic 就绪状态 |
| `--verbose` | `false` | 输出扫描诊断信息 |
| `--format` | `text` | 输出格式：`text` 或 `json` |
| `--no-color` | `false` | 禁用彩色输出 |

`--dry-run` 与 `--check` 互斥。

### `repository`

扫描 `internal/models/*.go`，为指定的 GORM 模型生成类型安全仓储。

**产物**

| 产物 | 路径 | 说明 |
|---|---|---|
| interface | `internal/repository/<model_snake>.go` | `Create / List / FindByID / Update / UpdateFields / Delete` 接口 |
| postgres 实现 | `internal/repository/postgres/<model_snake>.go` | GORM 实现，含 scope 过滤 |
| 模型 BeforeCreate hook | `internal/models/hooks.go` | 自动注入 `ensureUUID(&m.ID)`；首次创建时一并生成 `ensureUUID` 辅助函数 |

**Flag**

| Flag | 默认值 | 说明 |
|---|---|---|
| `-model` | （必填） | GORM 模型名 |
| `-label` | 同 `-model` | 人类可读标签，用于错误消息 |
| `-scope` | 自动推断 | 用户作用域，格式 `local_column:table.column:user_column` |
| `-root` | `.` | 项目根目录 |
| `--dry-run` | `false` | 预览模式 |
| `--check` | `false` | 过期检查模式 |
| `--list-models` | `false` | 列出所有模型及其 scope 状态（`direct` / `requires_scope` / `unsupported`） |
| `--verbose` | `false` | 输出扫描诊断信息 |
| `--format` | `text` | 输出格式 |
| `--no-color` | `false` | 禁用彩色输出 |

**Scope 规则**

- 模型拥有 `UserID uuid.UUID` 字段 → 直接作用域，`WHERE user_id = ?`
- 模型没有 `UserID` → 必须通过 `-scope` 声明父级作用域
  - 格式：`local_column:table.column:user_column`
  - 示例：`-scope conversation_id:conversations.id:user_id`
  - 校验：`local_column` 必须是模型实际字段
  - 行为：Create 时校验归属，List / Find / Update / Delete 时 `JOIN + WHERE` 过滤

### `docs`

扫描路由注解与 DTO tag，生成 `docs/openapi.json`（OpenAPI 3.0.3）。运行时仍通过 Gin 配置暴露。

**产物**

单文件 `docs/openapi.json`，包含：

- `openapi: 3.0.3`
- `info.title` / `info.version`
- `paths` — 所有路由，路径统一加 `/api` 前缀，Gin `:param` 转 OpenAPI `{param}`
- `components.securitySchemes.BearerAuth` — `BearerAuth` JWT 方案
- `components.schemas` — 自动提取 request / response DTO 结构

**Flag**

| Flag | 默认值 | 说明 |
|---|---|---|
| `-root` | `.` | 项目根目录 |
| `--output` | `docs/openapi.json` | 输出路径 |
| `--title` | `Cove API` | `info.title` |
| `--version` | `0.1.0` | `info.version` |
| `--dry-run` | `false` | 预览模式 |
| `--check` | `false` | 过期检查模式 |
| `--verbose` | `false` | 输出扫描诊断信息 |
| `--format` | `text` | 输出格式 |
| `--no-color` | `false` | 禁用彩色输出 |

**OpenAPI 映射规则**

| 注解 / DTO tag | OpenAPI 产物 |
|---|---|
| `@auth` | `security: [{BearerAuth: []}]` |
| `@summary` | `summary` |
| `@description` | `description`（多行拼接） |
| `@tag` | `tags` |
| `@input request.Xxx` + JSON body | `requestBody: application/json` |
| `@input request.Xxx` + multipart | `requestBody: multipart/form-data` |
| `@input request.Xxx` + `uri:"id"` | `parameters: in=path` |
| `@input request.Xxx` + query/GET | `parameters: in=query` |
| `@response XxxResponse` | `200 responses: {code, message, data}` envelope |
| `@sse + @event` | `200 responses: text/event-stream` |
| DTO `binding:"required"` | `required` 数组 |
| DTO `binding:"min=N"` / `max=N` | `minLength` / `maxLength` |
| DTO `binding:"email"` | `format: email` |
| `[]T` 类型 | `type: array, items: ...` |
| `ListResponse[T]` | `{list: [...]}` |
| `PageListResponse[T]` | `{total, page, page_size, list}` |

### `prompt`

扫描 `internal/prompts/prompts.go` 的 `registeredPrompts()`，解析每个已注册模板的
Go template AST，并按 namespace 生成 `internal/prompts/promptsgen/agent_gen.go`、
`memory_gen.go` 等文件。

生成器会把普通字段推断为 `string`，把 `range` 和 `join` 参数推断为切片，并为
嵌套字段生成独立结构体。类型冲突、动态 `index`、无法追踪的 range 表达式会直接
返回错误，不生成 `any`。

```bash
go run ./cmd/codegen prompt
go run ./cmd/codegen prompt --check
go run ./cmd/codegen prompt --dry-run --verbose
```

| Flag | 默认值 | 说明 |
|---|---|---|
| `-root` | `.` | 项目根目录 |
| `--output` | `internal/prompts/promptsgen` | 生成文件目录 |
| `--dry-run` | `false` | 预览模式 |
| `--check` | `false` | 检查生成文件是否与注册表及模板同步 |
| `--verbose` | `false` | 输出扫描到的提示词数量 |
| `--format` | `text` | 输出格式 |
| `--no-color` | `false` | 禁用彩色输出 |

### `doctor`

只扫描并报告协议、DTO、repository scope 等问题，不生成代码。输出 `Diagnostics` 列表，每条含 `level`（`error`/`warn`/`info`）、`code`、`message`、`hint`。

**诊断项**

| Code | Level | 含义 |
|---|---|---|
| `route.sse.missing_event` | error | 路由使用 `@sse` 但缺少 `@event` |
| `route.response.missing` | warn | 路由没有 `@response`/`@output` |
| `route.input.unknown_dto` | warn | `@input` 引用的 request DTO 未找到 |
| `route.uri.multiple_embedded` | error | request DTO 含多个 embedded URI-only 子结构 |
| `route.uri.missing_field` | error | Gin 路径参数在 request DTO 中缺失对应的 `uri` tag |
| `repository.id.required` | error | 模型缺少 `ID uuid.UUID`，无法生成仓储 |
| `repository.scope.required` | warn | 模型没有 `UserID uuid.UUID`，需要 `--scope` |

**Flag**

| Flag | 默认值 | 说明 |
|---|---|---|
| `-root` | `.` | 项目根目录 |
| `--verbose` | `false` | 输出扫描计数等诊断信息 |
| `--format` | `text` | 输出格式 |
| `--no-color` | `false` | 禁用彩色输出 |

| Flag | 默认值 | 说明 |

## 路由注解协议

### `@` 指令（推荐）

使用 `@` 前缀，单行独立：

```go
// @auth(user_id)              // 需要认证并提取 UserID
// @description 重命名会话      // OpenAPI description（多行拼接）
// 修改当前用户拥有的会话标题。 // description 后续行
// @summary 重命名会话          // OpenAPI summary
// @input request.RenameConversationRequest
// @response ConversationResponse
conversationRoutes.PATCH("/:conversation_id", conversation.RenameConversation)
```

SSE 路由必须声明事件类型：

```go
// @auth(user_id)
// @sse
// @event types.AgentEvent
// @input request.ChatStreamRequest
chatRoutes.POST("/stream", chat.ChatStream)
```

### `@routegen` 多行协议

以 `@routegen` 起始，后续 `@` 指令放在同一注释块：

```go
// @routegen
// @auth
// @userID
// @input request.CreateBookRequest
// @output response.ListResponse[*response.BookResponse]
bookRoutes.POST("", book.Create)
```

### `routegen:` 旧协议（继续兼容）

```go
// routegen: auth user_id input=request.CreateBookRequest output=response.BookResponse
bookRoutes.POST("", book.Create)
```

### 协议对照

| 功能 | `@` / `@routegen` | `routegen:` |
|---|---|---|
| 需要认证 | `@auth` | `auth` |
| 提取 UserID | `@auth(user_id)` 或 `@userID` | `auth user_id` |
| 请求体 | `@input request.Xxx` | `input=request.Xxx` |
| 响应 | `@response Xxx` | `output=Xxx` |
| SSE | `@sse` + `@event Type` | `sse ... event=Type` |
| 描述 | `@description ...` | 无（使用注释组上方文字） |
| 摘要 | `@summary ...` | 无 |

## Handler 请求绑定规则

工具根据 HTTP method 与 request DTO tag 自动选择 Gin 绑定方式：

| 条件 | 绑定方式 |
|---|---|
| GET + 无 URI 参数 | `ShouldBindQuery` |
| GET + 有 URI 参数 | `ShouldBindUri` + `ShouldBindQuery` |
| 非 GET + DTO 含 `*multipart.FileHeader` | `ShouldBind`（multipart） |
| 非 GET + DTO 含 JSON body | `ShouldBindJSON` |
| DTO 全部字段都是 URI-only | `ShouldBindUri`（不触发 JSON 绑定） |
| 非 GET + composite DTO（含 embedded URI-only 子结构） | `ShouldBindUri(&body.EmbeddedURIOnly)` + `ShouldBindJSON(&body)` |

## Report 格式

`Report` 结构为 JSON 友好设计，`--format json` 时可直接消费。

```json
{
  "command": "route",
  "mode": "write",
  "files": [
    {"kind": "added",      "path": "internal/transport/http/handler/book.go"},
    {"kind": "added",      "path": "internal/logic/book/create.go"},
    {"kind": "skipped",    "path": "internal/logic/book/list.go"},
    {"kind": "modified",   "path": "internal/models/hooks.go"},
    {"kind": "would-add",  "path": "..."},
    {"kind": "would-modify","path": "..."},
    {"kind": "unchanged",  "path": "..."}
  ],
  "diagnostics": [
    {
      "level": "error",
      "code": "route.sse.missing_event",
      "message": "ChatHandler.ChatStream uses @sse but missing @event <GoType>",
      "hint": "add @event types.AgentEvent or another SSE event type",
      "file": ""
    }
  ]
}
```

| FileChangeKind | 含义 |
|---|---|
| `added` | 新文件已写入 |
| `modified` | 已有文件已更新 |
| `would-add` | dry-run / check 模式下会新增 |
| `would-modify` | dry-run / check 模式下会修改 |
| `skipped` | 文件已存在（对应生成策略跳过） |
| `unchanged` | 格式化后内容一致，无需写入 |


## 生成产物示例

### Route 示例

输入 `routes/book.go`：

```go
// 创建图书。
// 保存图书的基础信息。
// @auth(user_id)
// @input request.CreateBookRequest
// @response BookResponse
bookRoutes.POST("", book.Create)
```

产物 1 — `internal/transport/http/handler/book.go`（新建时）：

```go
package handler

import (
	booklogic "github.com/boxify/api-go/internal/logic/book"
	"github.com/boxify/api-go/internal/svc"
	"github.com/boxify/api-go/internal/transport/http/request"
	"github.com/boxify/api-go/internal/transport/http/response"
	"github.com/boxify/api-go/internal/xerr"
	"github.com/gin-gonic/gin"
)

type BookHandler struct {
	svc *svc.ServiceContext
}

func NewBookHandler(svcCtx *svc.ServiceContext) BookHandler {
	return BookHandler{svc: svcCtx}
}

func (h BookHandler) Create(c *gin.Context) {
	var body request.CreateBookRequest
	if err := c.ShouldBindJSON(&body); err != nil {
		response.FromError(c, xerr.Validation(err))
		return
	}
	userID, err := util.UserIDFromContext(c.Request.Context())
	if err != nil {
		response.FromError(c, err)
		return
	}
	out, err := booklogic.NewCreateLogic(c.Request.Context(), h.svc).Create(userID, &body)
	if err != nil {
		response.FromError(c, err)
		return
	}
	response.OK(c, out)
}
```

产物 2 — `internal/repository/postgres/conversation.go`：

```go
// Code generated by codegen; DO NOT EDIT.

package postgres

import (
	"context"
	"errors"

	"github.com/boxify/api-go/internal/models"
	"github.com/boxify/api-go/internal/repository"
	"github.com/boxify/api-go/internal/xerr"
	"github.com/google/uuid"
	"gorm.io/gorm"
)

type ConversationRepository struct {
	db *gorm.DB
}

func NewConversationRepository(db *gorm.DB) repository.ConversationRepository {
	return &ConversationRepository{db: db}
}

func (r *ConversationRepository) Create(ctx context.Context, userID uuid.UUID, conversation *models.Conversation) (*models.Conversation, error) {
	conversation.UserID = userID
	if err := r.db.WithContext(ctx).Create(conversation).Error; err != nil {
		return nil, xerr.Wrapf(err, "创建会话失败")
	}
	return conversation, nil
}

```
